Positions


La propiedad "position", especifica el tipo de metodo de posicionamiento usado para un elemento.

Tiene 5 tipos:

"static" : Los elementos posicionados estáticos no se ven afectados por las propiedades superior, inferior, izquierda y derecha.
Un elemento con este posicionamiento no está posicionado de ninguna manera especial; siempre se coloca de acuerdo con el flujo normal de la página


"relative" : Este posicionamiento, se ajusta dependiendo la disponibilidad de espacio.
Al establecer las propiedades superior, derecha, inferior e izquierda de un elemento relativamente posicionado hará que
se ajuste fuera de su posición normal. Establecer las propiedades superior,
derecha, inferior e izquierda de un elemento relativamente posicionado hará que se ajuste ajustarse lejos de su posición normal.


"fixed": Se coloca en relación con la ventana gráfica, lo que significa que siempre permanece en el mismo lugar,
incluso si se desplaza la página. Las propiedades superior, derecha, inferior e izquierda se utilizan para colocar el elemento.


"absolute": Se coloca en relación con la entidad posicionada más cercano
(en lugar de posicionarse en relación con la ventana gráfica, como fijo).
Sin embargo, si un elemento con posición absoluta no tiene entidad padre con otra posición,
utiliza el elemento "body" del documento.


"sticky": Los elementos con esta posoción, se colocan en la parte superior,
dependiendo de la barra del desplazamiento. Es mejor que el nav tenga este posicionamiento

Transiciones


El valor Transition, es un acompañamiento del hover

Como la mayoria de de las etiquetas que hemos visto hasta ahora,
tiene tanto configuracion general como individual


Estas son o "transition", o también:
"transition-delay" (tanto tiempo hasta el comienzo de la animacion)
"transition-duration" (duracion de la transicion)
"transition-property" (defines a que elemento solo le afecta la transicion: width, height, padding...)
"transition-timing-function" (especifica la velocidad de la curva de la animacion,
estos son: linear, ease, ease-in, ease-out, ease-in-out)


La propiedad "transition", en si misma, ejecuta todas las anteriores si es deseado, el unico inconveniente es que si tenemos "transition"
no podemos añadir ninguna de las individuales, pero, si no esta podemos añadir todas las individuales

Animaciones


Las animaciones, son como los hover, pero mas avanzados, es el siguiente nivel.

CSS, te permite dar "vida" a los elementos HTML sin necesidad de JavaScript o Flash

La animacion siempre va dentro de "@keyframes", y tiene que tener 2 cosas, un inicio y un final.

Al igual que con "transition", "animation", tambien tiene formas individuales. Estas son:


"animation-name": El nombre al cual se le añadira el "@keyframes".

"animation-duration": La duracion de la animación.

"animation-delay": Retardo de la animación.

"animation-iteration-count": Numero de veces que se repetira la animación.

"animation-direction": Indica si la animacion irá hacia adelante, reversa o en ciclos alternativos.

"animation-timing-function": Indica la velocidad de la curva de la animacion (linear, ease...).

"animation-fill-mode": Especifica un estilo, para el elemento deseado, cuando la animacion no esta
siendo ejecutada.


La construccion del general es "animation: [NombreAnimación] [Duración] [FunciónTiempo] [Retraso] [Iteraciones] [Dirección]"

Botones & Pseudoclase Active


"Button" es un elemento interactivo activado por el usuario.
Una vez activo, ejecuta una acción preprogramada.


Existen 3 tipos:
"button": el cual sirve para ejecutar acciones preprogramadas
"a": se usa para crear botones con links externos
"input" (type="button"): el cual sirve para completar formularios o enviar información


La pseudo-clase ":active", representa un elemento que el usuario esta activando.
Cuando se usa un mouse, la "activación" generalmente comienza cuando el usuario presiona el botón primario del mouse y
termina cuando se suelta. La pseudo-clase :active se usa comúnmente en los elementos "a" y "button", pero también se puede usar en otros elementos

Combinadores


Los combinadores, es algo que explica la relacion entre selectores


Un selector de CSS, puede contener más de un simple selector, y entre ellos podemos poner un combinador.
Existen 4 tipos:

Selector descendiente (espacio): Este "selecciona" todos los elementos que son descendientes de un elemento en especifico.

Selector de hijo (>): Selecciona todos los elementos hijos de un elemento en especifico.

Selector de hermano adyacente (+): Es utilizado para seleccionar UN elemento distinto y continuo al mismo "nivel".
(Misma cantidad de tabulación si eres ordenado)

Selector de hermano general (~): Es utilizado para seleccionar VARIOS elementos distintos y continuos al mismo "nivel".

Las medidas absolutas, no son recomendadas para utilizar en pantalla, debido a que el tamaño de ellas varia mucho

que puedes crear variables, con rango global o local, cambiarlas con JS,
y cambiar las variables basado en los "media queries".


La sintaxis es simple: "--[nombre]:valor;"
En valor, puedes poner el dato que quieras, claro esta, que luego tienes
que saber que valor tiene asginado
